# Sprint Plan — 2026-05-17 (E-commerce Project Repositioning)

> Companion document for the weekend sprint. Sibling plan lives at [`saas-product-analytics/docs/sprint_2026_05_17.md`](https://github.com/Leo309/saas-product-analytics/blob/main/docs/sprint_2026_05_17.md).

## Context

This repository is one of two portfolio projects targeted by the 2026-05 weekend sprint. The original implementation was built on **Microsoft Fabric** (Lakehouse + DirectLake) with **PySpark** transformations and a **Power BI** semantic model dashboard. The Fabric workspace license has expired, which means the dashboard is no longer live and the workspace can't be re-entered without paying for a new Azure subscription.

The honest move is **not to pretend this is a live project**. The sprint goal for this repo is therefore **repositioning, not rebuilding**: turn this into a defensible "code archive of a real Fabric implementation" with a README that explains exactly what was built, what's preserved, and what isn't.

## Scope Decision

**In scope today**:
- Rewrite `README.md` as an honest case study (Fabric implementation, license expired, code preserved)
- Write `docs/original_fabric_implementation.md` that consolidates the 21 Power BI screenshots, Fabric architecture, and design decisions
- Both are documentation-only; **no code changes**

**Out of scope (deliberately not done)**:
- Rebuilding the pipeline on an open-source stack (DuckDB / Delta Lake locally)
- Replacing Power BI with Looker Studio for a live dashboard
- Adding GitHub Actions CI for this repo

**Rationale**: The keywords this repo carries (PySpark / Fabric / Power BI / Medallion / Star Schema / Azure) are already covered by the existing 960 lines of PySpark code and 21 Power BI screenshots — no code change improves keyword coverage. CI in this repo would duplicate the CI/CD story already implemented in the sibling SaaS project, and PySpark code can't actually be exercised locally without significant rework that isn't justified by the weekend budget.
